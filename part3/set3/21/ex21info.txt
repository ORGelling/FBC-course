Exercise 21: Compare types of template parameters

We use recursion to bite off one one parameter from haystack using specialised
templates, ending either in a found (with a param pack that can either hold
spare untested args or just be empty) or a not found (at all) case.

We have used a class rather than a struct this time purely for aesthetic
reasons to avoid having to set extra private and public sections, as we use a
private enum (next) to check the n ext entry, and then keep track of the final
result using locate. Not keeping these two separate led to issues with
returning positive values after multiple cycles, even if no matching type was
eventually found.
