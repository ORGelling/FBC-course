Exercise 21: Compare types of template parameters

We use recursion to bite off one one parameter from haystack using specialised
templates, ending either in a found (with a param pack that can either hold
spare untested args or just be empty) or a not found (at all) case.

We note that using the TypeIdx class here seems entirely superfluous. One can
compute the index without even needing to hand an index inbetween the
recursive calls. The exact same structure can be used as done here, but all
within Type itself. The amount of recursion iterations count the index
already, without ever having to hand a "value"down. This was my first solution
but I had to amend it to add a TypeIdx class (struct) and separate this
functionality out. The idx non type is here only because the exercise asks it
of us. Compact solution is added as Type2.
