Exercise 43: design a scanner scanning a piece of text

We have implemented the scanner to read and write directly from and to files,
using their names from the commandline. We do not have the setup ready to use
a temporary name, remove the original, and the change the temp name to the
original's. That is something that is likely best left to the main program,
and we are asked here to hand in the scanner contents only. 

Setting up this mechanism would mean, as the exercise implies, writing a small
class that finds a fitting temporary file name, and then using that when
calling the scanner, before running it and then removing the scanned file and
changing the name of the temporary.

We are happy with a scanner that can read from and write to files directly
already, and does not need any further instructions or while-switches to
process the contents of the files.
