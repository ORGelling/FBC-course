Exercise 61: Use parsing in non-standard situations

We use the format of irrelevant lines, being a comand of some type + a path
name, to allow the grammar to swallow these lines without doing anything. The
relevant lines, starting with var:, fun:, or expr:, are given their own paths
that actually check their syntax and make sure variables and functions are
(not multiply) defined before use. We show error messages depending on what 
was wrong on the line, this does mean we sometimes show multiple error 
messages per line, if there were multiple errors. This only happens if there 
was no syntax error, but an un-, or multiply, defined function/variable.

The output from our program for a file test.txt:

logname  /var/log/program.log
socket   /var/run/program.sock
pidfile  /var/run/program.pid
database ~/program/data.db

    # define the variable 'sum'
var: sum = 0    
    # a function 'sumFun()' can be called
fun: sumFun()
    # expr: is followed by a valid expression
expr: sum += someFun() + 12;
expr: some -= sumFun() - 3;

fun: 5

is the following:

At line 11: Function not declared: someFun()
At line 12: Variable not declared: some
At line 14: Syntax error
