Exercise 49: specify a simple grammar

We have specified a very simple grammar indeed, that can parse variable names
and numerical values inside write() statements. The exercise mentions that
Pascal's write() function will print the value that variables hold, our
grammar does not. Keeping track of the mentioned variables and printing their
value rather than variable name is a lot more complex, so for now we will keep
it at this.

Our grammar should now be restrictive enough to now allow commas in the wrong
place, having a (left) recursive pattern for write_elements_list that can be
appended with ',' and another element indefinitely.

We note that we also have our parser check for the ';' that ends the write()
statement, even though that is not exactly necessary. If this is outside the
scope of the parser then it would be fine to have the scanner ignore this
character and the parser only need a closing bracket for the write() rule to
be valid.
